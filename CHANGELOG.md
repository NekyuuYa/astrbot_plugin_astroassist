# 更新日志 (CHANGELOG)

## [v0.5.4] - 2026-04-02
### 修复
- **渲染回归**：回归 HTML 渲染引擎，以解决 Pillow 绘图带来的字体缺失和复杂的布局对齐问题。
- **超清补偿**：通过 1000px 视口 + 3x 采样率，输出 3000 像素宽度的超高清图片，对抗平台模糊。
- **传输增强**：显式构造 `base64://` 协议字符串，确保适配器能够稳定解析图片数据。

## [v0.5.3] - 2026-04-02
### 变更
- **引擎彻底重构**：切换至自主实现的 Pillow (PIL) 像素级渲染引擎，彻底移除对浏览器（Playwright/Chromium）的依赖。
- **高清重制**：默认采用 2 倍物理超采样，渲染宽度达 1200px，提供绝对清晰的文字和图形边缘。
- **M3 还原**：手动还原了 Material Design 3 风格，包括圆角矩形、进度填充和 Tonal 配色。
- **稳定性**：通过 Base64 编码和零外部依赖方案，解决所有环境下的图片生成和传输失败问题。

## [v0.5.2] - 2026-04-02
### 修复
- **传输稳定性**：改用 Base64 编码传输图片，解决 Docker/跨环境下的 `rich media transfer failed` (1200) 错误。
- **裁剪优化**：移除卡片外边距，实现 HTML 渲染后的像素级紧凑裁剪。

## [v0.5.1] - 2026-04-02
### 优化
- **渲染精度**：回归 Playwright 内核，显式指定 `type: "png"` 实现无损截图。
- **高清视口**：采用 1200px 逻辑宽度配合 `scale: "device"` 超采样，彻底解决模糊问题。
- **布局修正**：通过 `display: inline-block` 消除边缘空白，并优化 Material Design 3 工业排版。

## [v0.5.0] - 2026-04-02
### 变更
- **引擎重构**：弃用内置 html_render，改用原生 Pyppeteer (Puppeteer) 驱动 Chromium。
- **高清策略**：设置 `deviceScaleFactor: 3`，实现 3x 超采样渲染，彻底解决图片模糊问题。
- **样式微调**：进一步优化 Material Design 3 布局密度。

## [v0.4.6] - 2026-04-02
### 优化
- **物理精度提升**：在 HTML 模板中引入 CSS `zoom` 物理缩放，强迫浏览器以更高密度绘制矢量图形。
- **视口调整**：加大渲染视口宽度以配合物理缩放，彻底消除模糊感。

## [v0.4.5] - 2026-04-02
### 修复
- **缩进与语法**：修复了 main.py 中的缩进错误。
- **渲染优化**：进一步微调 HTML 模板以确保 3600px 高清输出下样式一致性。

## [v0.4.4] - 2026-04-02
### 优化
- **渲染清晰度**：将基础渲染宽度提升至 1200px，并配合 `device_scale_factor: 3`，实现 3600px 物理宽度的超清输出。
- **发送策略**：改为从文件系统直接读取并发送本地渲染图，绕过 URL 转发可能导致的二次压缩。

## [v0.4.3] - 2026-04-02
